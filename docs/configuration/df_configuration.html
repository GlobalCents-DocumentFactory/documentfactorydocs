Configuration

DocumentFactory relies on a configuration file named .env located in the Installation Directory, which is specified during the installation process (referred to as Select the Installation Location). This file defines essential runtime parameters that control the behavior of various components such as workflow processing, object storage, and messaging.

It follows the standard dotenv format, where each line defines a key-value pair. Below is a sample .env file with default parameter values:

.env
DOTNET_ENVIRONMENT="Production"

PTFD_MAX_CONCURRENTWORKFLOWS=20
PTFD_MAX_CONCURRENTTASKS=40
PTFD_TASKS_SECONDSTIMEOUT=600
PTFD_WORKFLOWS_SECONDSTIMEOUT=6000

S3_SERVER_START=false
S3_HOSTPORT=":9000"
S3_CONSOLEHOSTPORT=":9001"
S3_ROOT_USER="documentfactory"
S3_ROOT_PASSWORD="secretpassword"

MINIO_USE_MINIO=false
MINIO_URL=localhost:9000
MINIO_USE_SSL=false
MINIO_REGION="us-east-1"

NATS_SERVER_START=true
NATS_URL="nats://127.0.0.1:4222"
NATS_PUBLICKEY="UAWEEECHAJZUZGOOOI6N5TDJK7ZHXYG5IW2SZ6KK7EVHAXAK64RC4ICR"
NATS_PRIVATEKEY="SUAMCB5PRHKKQ44QS674C3UDVUHF2RTPTT2JM3KW4KMDKLYVMN2XQNQYVQ"
NATS_SERVERNAME="nats-server"
NATS_INMEMORY=true
NATS_SERVER_ROOTFS="C:\\DocFactory\\DocFactoryData"

Configuration sections:

  • Environment

    • DOTNET_ENVIRONMENT specifies the environment in which DocumentFactory is running (e.g., Development, Staging, or Production). It affects configuration loading, logging levels, and component behavior at runtime. In most cases, this option should be left at its default value unless there is a specific need to change the deployment environment.

  • Workflow Processing Settings

    • PTFD_MAX_CONCURRENTWORKFLOWS - defines the maximum number of workflows that can be processed in parallel. Higher values may improve throughput but require more system resources.

    • PTFD_MAX_CONCURRENTTASKS - sets the maximum number of concurrent tasks across all workflows. Useful for controlling system load when tasks are resource-intensive.

    • PTFD_TASKS_SECONDSTIMEOUT - specifies the maximum time (in seconds) a single task is allowed to run before being terminated.

    • PTFD_WORKFLOWS_SECONDSTIMEOUT - defines the timeout (in seconds) for an entire workflow. Ensures stuck or long-running workflows do not block the system indefinitely.

  • MinIO Integration: If using MinIO as an external object store, configure its connection parameters here.

    • S3_SERVER_START – Enables or disables the embedded S3-compatible server. Set to true to start the server automatically with DocumentFactory.

    • S3_HOSTPORT – The network address (host and port) the S3 server will bind to (e.g., :9000 for all interfaces on port 9000).

    • S3_CONSOLEHOSTPORT – The address for accessing the MinIO web-based console (e.g., :9001).

    • S3_ROOT_USER – Specifies the root access key used to authenticate requests to the built-in S3-compatible server. See How to Generate Access Keys for instructions on creating access credentials.

    • S3_ROOT_PASSWORD – Defines the root secret key associated with the access key above. It is used to securely sign requests and must be kept confidential. Refer to How to Generate Access Keys for the generation process.

    • MINIO_USE_MINIO – Enables or disables the use of MinIO as the object storage backend. Set to true to activate MinIO integration.

    • MINIO_URL – The endpoint address where the MinIO server is running (e.g., localhost:9000).

    • MINIO_USE_SSL – Set to true if MinIO is configured to use HTTPS; otherwise, set to false for plain HTTP.

    • MINIO_REGION – Defines the region for MinIO. While MinIO accepts any region string, the default is typically "us-east-1". This value may be important for compatibility with S3 clients or services expecting a region tag.

  • NATS Messaging: Controls whether a NATS server should be started with DocumentFactory and how to connect to it.

    • NATS_SERVER_START – If set to true, DocumentFactory automatically starts an embedded NATS server on launch.

    • NATS_URL – The address used to connect to the NATS server (typically nats://localhost:4222).

    • NATS_PUBLICKEY - The public NKey used to identify the client to the NATS server. It is included in the authentication process and is matched against the server’s list of allowed keys to determine access permissions (e.g., publish, subscribe). These should be generated using the nk tool. For more details, see the (key generation instructions).

    • NATS_PRIVATEKEY – The private NKey seed used to cryptographically sign authentication challenges issued by the NATS server. This proves the client owns the corresponding public key without exposing the private key itself. This key is highly sensitive and must be kept secret. For more details, see the (key generation instructions).

    • NATS_SERVERNAME – The expected server name for identity verification.

    • NATS_INMEMORY – Determines the storage backend for NATS JetStream. If set to true, document processing state and message streams are stored in memory using MemoryStorage. If set to false, the client uses file-based storage (FileStorage), persisting data to disk. This setting influences performance and durability trade-offs.

    • NATS_SERVER_ROOTFS – The directory path used by the NATS server to store persistent data when not using in-memory mode.
