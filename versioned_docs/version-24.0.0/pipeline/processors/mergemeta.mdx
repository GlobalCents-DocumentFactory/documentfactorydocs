# Merge Meta

## `actionmergemeta`

The `actionmergemeta` property activates the metadata merging feature within the document processing pipeline.
Setting this property to `true` within the `processors` or `postprocess` arrays enables the build process to inspect
the documents and replace the [**metadata tags**](docs/pipeline/metadata.mdx). If this property is set to `false`, no replacements will take place.

## `settingsmergemeta`

The `settingsmergemeta` property provides configuration parameters for the metadata merging action.
The `settingsmergemeta` property can contain the following properties:

- <Link to="#mergenullorempty">`mergenullorempty`</Link> - controls whether metadata fields that are null or empty
    should be included in the merging process.
- <Link to="#removeemptylines">`namingpattern`</Link>
- <Link to="#usestrictreplacement">`settingswordmerge`</Link>
- <Link to="#wordtrackchanges">`saveformat`</Link>

### `mergenullorempty`

Property `mergenullorempty` controls whether metadata fields that are null or empty should be included
in the merging process.
To optimize the process by excluding irrelevant data and retaining tags in the output document if they are invalid or empty,
set this property to `false`. This skips the merging for null or empty metadata fields.
Alternatively, to ensure that all metadata fields are included, even if they have null or empty values, set this property to `true`.
This option processes all tags in the replacement list and removes any invalid replacement tags from the source document.
